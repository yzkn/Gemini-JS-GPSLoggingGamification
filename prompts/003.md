# システム要件定義書：高速道路RTAアプリ（Racing Dashboard Edition）

## 1. アプリケーション概要
*   **タイトル:** 高速道路RTAアプリ
*   **コンセプト:** 高速道路のチェックポイント（CP）通過タイムを計測する、レーシングゲーム風UIのWebアプリ。
*   **動作環境:** スマートフォン（Webブラウザ/PWA）
*   **前提:** フォアグラウンドでの常時表示（Wake Lock APIを使用）。

## 2. デザイン・UI/UX要件（レーシング・ダッシュボード）

### 2.1 ビジュアルコンセプト
*   **配色:** `bg-slate-950`（深い紺〜黒）を基調とし、文字はネオンカラーを採用。
    *   **計測中:** ネオンブライトグリーン（`text-green-400`）
    *   **一時停止:** ネオンオレンジ（`text-orange-400`）
    *   **警告/未開始:** ネオンレッド（`text-red-500`）
*   **フォント:** デジタルフォント（または視認性の高いMonospaceフォント）。

### 2.2 画面レイアウト構成
1.  **ヘッダー（情報集約エリア）:**
    *   **最優先表示:** 次のCPまでの「直線距離（km）」を巨大なフォントで中央に配置。
    *   **進捗バー:** 1つ前のCPから次のCPまでの距離を100%とした現在地の進捗をバーで視認可能にする。
2.  **メイン（地図エリア）:**
    *   **MapLibre GL JS:** ベースマップは地理院タイル（夜間は淡色/ダークトーンへ自動切替）。
    *   **CPピン表示:** ハードコードされたCP座標を地図上にピン（マーカー）として常時表示。
    *   **走行軌跡:** 測位結果をグロー効果（ネオンライン）付きの直線で結ぶ。
3.  **サイドオーバーレイ（ラップリスト）:**
    *   右側に半透明のブラックパネルを配置し、通過済みCPのラップタイムを最新順に表示。
4.  **下部（コントロール/サブ情報）:**
    *   **時速:** `coords.speed` をデジタル表示。
    *   **ボタン:** 「計測開始」「計測終了」「一時停止」を配置。
    *   **誤操作防止:** 「計測終了」は誤タップ防止のため長押しまたはスライド操作とする。

### 2.3 視認性・UX向上機能
*   **Wake Lock API:** 起動時の「安全警告画面」でのユーザー操作をトリガーにスリープを防止。
*   **ダイナミック・ズーミング:** 次のCPまでの距離が500mを切った場合、地図を自動でズームアップする。
*   **音声読み上げ (Web Speech API):** CP通過確定時、「[CP名]を通過しました」と読み上げ。
*   **触覚フィードバック (Vibration API):**
    *   CP通過：短い2回の振動
    *   計測開始：長い1回の振動
    *   GPSロスト（精度低下）：連続した短い振動

## 3. ロジック仕様

### 3.1 測位・フィルタリング
*   **測位頻度:** 5秒間隔 または 移動距離5m以上。
*   **外れ値判定:**
    *   前回測位地点からの速度換算が 180km/h を超える場合は破棄。
    *   `accuracy`（精度）が100m以上の場合は破棄。
*   **移動平均:** 地図表示用プロットのみ直近3点の移動平均を使用。IndexedDB保存と時速表示は生データを使用。

### 3.2 チェックポイント通過判定
*   **判定アルゴリズム:** CP座標から半径50m以内に入った後の「最短距離地点」を通過時刻とする。
*   **確定条件:** 最短距離を記録した後、再びCPから離れ始めた瞬間に「通過」と確定させる。
*   **補正（GPS揺らぎ対策）:** 「離れた」と判定した後でも、半径1km以内であれば、より短い距離が観測された場合に記録を上書き更新する。1kmを超えたら次のCPへ移行。
*   **トンネル補完:** GPSロスト地点と復帰地点の間に未通過CPがあった場合、その間の時間を距離で等間隔に割って（線形補完）通過時刻を算出する。

## 4. データ管理

### 4.1 CP定義（ハードコードJSON例）
```json
[
  {
    "id": "kan-etsu-down",
    "routeName": "関越川越（下り）",
    "checkpoints": [
      { "name": "川越IC", "latitude": 35.893205, "longitude": 139.459547 },
      { "name": "三芳PA", "latitude": 35.844391, "longitude": 139.503090 },
      { "name": "所沢IC", "latitude": 35.809777, "longitude": 139.529614 }
    ]
  },
  {
    "id": "kan-etsu-up",
    "routeName": "関越川越（上り）",
    "checkpoints": [
      { "name": "所沢IC", "latitude": 35.807816, "longitude": 139.530606 },
      { "name": "三芳PA", "latitude": 35.839042, "longitude": 139.503693 },
      { "name": "川越IC", "latitude": 35.892501, "longitude": 139.459501 }
    ]
  }
]
```

### 4.2 エクスポート仕様
*   **CSV:** 日時、CP名、ラップ、スプリット、全ログ座標（経緯度・時刻・速度）を1ファイルに混在。`type`列（LOG/CP）で識別。
*   **GeoJSON:** 1回セッションを1つの `LineString` として出力。

## 5. 使用技術・デプロイ
*   **Frontend:** Vite, Vanilla JS, TailwindCSS (npm install方式)
*   **Library:** MapLibre GL JS, geographiclib (高精度距離計算用)
*   **Storage:** IndexedDB
*   **PWA:** マニフェスト作成、地図タイルのService Workerキャッシュ（オフライン対応）
*   **Deployment:** Cloudflare Pages (Buildコマンド: `npm run build`)
